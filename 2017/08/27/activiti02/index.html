Activiti流程引擎开发（二）

发表于 2017-08-27 | 分类于 Activiti | 阅读次数
今天继续Activiti开发总结，将介绍BPMN基本知识、两种Activiti流程设计工具

BPMN的基本概念

   Business Process Modeling Notation,简称BPMN，译为业务流程建模标注。BPMN定义了业务流程图，其基于流程图技术，同时对创建业务流程操作的图形化模型进行了裁减。业务流程的模型就是图形化对象的图网，包括活动（也可以说工作）和定义操作顺序的流控制。
   BPMN2.0规范的分类划分为以下几个部分：

  • 启动与结束事件（Event）
  • 顺序流（Sequence Flow）
  • 任务（Task）：用户任务（UserTask）、系统任务（ServiceTask）
  • 网关（Gateway）：排他网关（ExclusiveGateway）、包容网关（InclusiveGateway）、并行网关（ParallelGateway）
  • 子流程（Subprocess）
  • 边界事件（Boundary Event）
  • 中间事件（Intermediate Event）
  • 监听器（Listener）:任务监听器（TaskListener）

启动事件

   启动事件分为三种类型：

  • 空启动事件
  • 定时启动事件
  • 异常启动事件

结束事件

   结束事件可以分为以下三种类型：

  • 空结束事件
  • 异常结束事件
  • 终止结束事件
  • 取消结束事件

顺序流

   顺序流可以细分为两种：

  • 标准顺序流
  • 条件顺序流

任务

   根据业务需求的不同分为以下类型：

  • 用户任务
  • 脚本任务
  • WebService任务
  • 业务规则任务
  • 邮件任务
  • Mule任务
  • Camel任务
  • 手动任务
  • Java service任务
  • Shell任务

网关

   网关（Gateway）用于控制流程走向（在BPMN2.0规范中称为“执行令牌”）。根据功能不同可以划分为以下4种网关：

  • 排他网关（ExclusiveGateway）
  • 并行网关（ParallelGateway）
  • 包容网关（InclusiveGateway）
  • 事件网关

子流程与调用活动

边界与中间事件

   除了提到的启动和结束事件之外，中间事件提供的特殊功能可以用来处理流程执行过程中抛出、捕获的事件，具体包括边界事件、中间捕获事件、中间抛出事件。

边界事件

   边界事件分为以下几种：

  • 定时器边界事件
  • 异常边界事件
  • 信号边界事件
  • 取消边界事件
  • 补偿边界事件

中间捕获事件

   中间捕获事件是流程中的“拦路虎”，根据事件的不同类型需要使用不同的方式才能继续执行后续的输出流的活动。例如执行完一个任务1天后执行下一个活动，或者一个指定的信号、消息被接收到之后再执行后续输出流的活动。分为以下三种：

  • 定时器中间捕获事件
  • 信号中间捕获事件
  • 消息中间捕获事件

中间抛出事件

   中间抛出事件和中间捕获事件是两个相互依赖的关系，中间捕获事件需要有时间抛出才能被触发，而中间抛出事件需要有对应的捕获事件接受才有意义。分为以下两种：

  • 空中间抛出事件
  • 信号中间抛出事件

监听器

   监听器分为两类：执行监听器和任务监听器。

执行监听器

   执行监听器允许在执行流程过程中执行Java代码（实现了监听器接口）或表达式。
   执行监听器可以捕获的事件如下：

  • 流程实例启动、结束
  • 输出流捕获
  • 活动启动、结束
  • 路由开始、结束
  • 中间事件开始、结束
  • 触发开始事件、触发结束事件

任务监听器

   相对于执行监听器的使用范围来说，任务监听器的使用范围要小很多，因为它只能应用于用户任务，用来坚挺3种事件。

  • create:在任务被创建且所有的任务属性设置完成之后才触发。
  • assignment:在任务被分配给某个办理人之后触发。有一点需要注意，assignment事件总是在create事件触发之前被触发，因为任务的办理人是一个属性，而create事件需要逐一处理任务的办理人、候选人、候选组属性。
  • complete:在配置了监听器的上一个任务完成时任务数据被删除之前（Activiti的数据分为运行时和历史两种类别，当运行时数据处理完毕后将会被清理）触发。

流程设计工具

   Activiti针对不同的平台提供了不同架构的流程设计器：

  • 基于B/S架构的流程设计器Activiti Modeler（可以通过浏览器在线设计流程）
  • 基于Eclipse插件的流程设计器Activiti Designer

Activiti Modeler

   在Activiti Explorer中使用Activiti Modeler




Activiti Designer

   基于Eclipse插件的流程设计器Activiti Designer

   两个设计器侧重面不同，Activiti Modeler偏向于业务层面，而Activiti Designer则更侧重于开发层。
